# ระบบจัดการยาง — สรุป Flow การทำงาน
### สาขาลาดกระบัง (ใช้ได้กับสาขาสระบุรีเช่นกัน)

---

## ภาพรวม Flow

```
รับยางเข้า Stock
      │
      ▼
Admin บันทึกยางเข้าระบบ (Stock)
      │
      ▼
คนขับพบปัญหายาง → ขอเปลี่ยนยางผ่าน App หรือ Web
      │
      ▼
Admin พิจารณาอนุมัติ → นัดหมายเข้าเปลี่ยน
      │
      ▼
ยางถูกเปลี่ยน → ข้อมูลอัปเดตกลับเข้า ATMS → Sync กลับมาที่ระบบ
```

## 1. Stock & PR Report
**`/tire/latkrabang/stock-tire`**

> บริหารจัดการสต็อกยางทั้งหมดของสาขา

### ทำอะไรได้บ้าง

**Tab: Stock**
- ดูยางทั้งหมดที่มีในระบบ — คงเหลือ / เบิกใช้แล้ว / ขายแล้ว
- กรองด้วย PR Code, ช่วงวันที่รับยาง (Deposit Date), สถานะ
- Summary cards: จำนวนยางทั้งหมด / In Stock / มูลค่ารวม / มูลค่า In Stock
- เพิ่มยางเข้าระบบ — ทีละเส้นหรือ Bulk วางหลายเส้นพร้อมกัน
- Export Excel สำหรับใช้ประกอบการอนุมัติจัดซื้อ

**Tab: รายงาน PR**
- เลือก PR Code เพื่อดูรายงานว่าล็อตนั้น **ยางแต่ละเส้นถูกนำไปใช้กับคันไหน**
- ข้อมูลต่อเส้น: ทะเบียนรถ / คนขับ / ตำแหน่งที่ใส่ / สาเหตุที่เปลี่ยน / รูปถ่าย
- ตัวชี้วัดประสิทธิภาพ: ระยะทางใช้งาน / % ประสิทธิภาพคงเหลือ / ฿ ต่อกิโลเมตร (จริง vs มาตรฐาน)
- สรุปภาพรวม PR: มูลค่ารวม / ระยะทางเฉลี่ย / สาเหตุหลักที่เปลี่ยน (แผนภูมิ)
- แชร์ลิงก์รายงานให้ผู้บริหารได้เลย

### ใช้เมื่อไหร่
- ก่อนสั่งซื้อยางล็อตใหม่ → ดูว่าล็อตเก่าคุ้มค่าไหม ยางยี่ห้อไหนคุ้มกว่า
- ตรวจสอบว่ายางในคลังเหลือเท่าไหร่ ต้องเติมหรือเปล่า
- ใช้ประกอบการขออนุมัติจัดซื้อยางเข้า Stock ใหม่

## 2. Change History (ประวัติการใช้ยาง)
**`/tire/latkrabang/change-history`**

> ประวัติยางทุกเส้นอ้างอิงจาก Serial No ดึงข้อมูลจาก ATMS

### ทำอะไรได้บ้าง
- ดูประวัติยางทุกเส้นทุกคัน — เปลี่ยนเข้าเมื่อไหร่ / เปลี่ยนออกเมื่อไหร่
- รู้ว่าปัจจุบันยางเส้นนั้น **อยู่คันไหน ตำแหน่งไหน**
- ติดตามสถานะล่าสุด: ยังใช้อยู่ / ถูกถอดออกแล้ว / ส่งซ่อม / ขาย
- ค้นหาด้วยทะเบียนรถ, Serial No, ชื่อสินค้า, เลขแจ้งซ่อม
- **Sync from ATMS** — ดึงข้อมูลล่าสุดจาก ATMS มาอัปเดตทันที
- **Auto-sync ทุกวัน 02:00** — ระบบดึงข้อมูลอัตโนมัติโดยไม่ต้องกดเอง

### ใช้เมื่อไหร่
- ตรวจสอบว่ายางเส้นที่มี Serial No นี้ถูกใช้กับคันไหนมาบ้าง
- ยืนยันว่ายางถูกเปลี่ยนเข้าจริงหรือเปล่าหลังจากอนุมัติแล้ว
- ดูประวัติย้อนหลังเพื่อวิเคราะห์อายุการใช้งาน

## 3. Change Tire Request (คนขับขอเปลี่ยนยาง)
**`/tire/latkrabang/change-tire-request`**

> คนขับกรอกคำขอเปลี่ยนยาง ระบบเทียบยางที่ใช้อยู่กับมาตรฐาน

### ขั้นตอนคนขับ
1. กรอกทะเบียนรถ + เลขไมล์ปัจจุบัน
2. ระบบดึงรายการยางที่ติดอยู่กับรถคันนั้น พร้อมแสดง:
   - ตำแหน่งยาง / Serial No / ยี่ห้อ
   - ระยะทางใช้ไปแล้ว / % ประสิทธิภาพคงเหลือ
   - ยางที่ใกล้หมดอายุจะแสดงสีแดงเตือน
3. เลือกยางเส้นที่ต้องการเปลี่ยน แล้วกรอก:
   - **สาเหตุ** — หมดดอก / ยางระเบิด / ยางฉีก / ยางบวม / รถกินยาง
   - **หมายเหตุ** — รายละเอียดเพิ่มเติม
   - **รูปถ่าย** — แนบได้สูงสุด 2 รูปต่อเส้น
4. กด Submit → คำขอส่งไปให้ Admin พิจารณา

### ส่งคำขอได้จาก
- 📱 **Mobile App** — คนขับกรอกจากในรถ
- 💻 **Admin Panel (mena-wms)** — เจ้าหน้าที่กรอกแทนได้

### ใช้เมื่อไหร่
- ยางมีปัญหา ต้องการเปลี่ยน
- ยางใกล้หมดอายุตามระยะทางมาตรฐาน
- ต้องการให้ Admin จัดสรรยางจาก Stock ให้

## 4. Requests (Admin อนุมัติ)
**`/tire/latkrabang/requests`**

> Admin ตรวจสอบและอนุมัติคำขอเปลี่ยนยาง

### ขั้นตอน Admin
| สถานะ | สีแถว | การดำเนินการ |
|-------|-------|------------|
| **Pending** | 🟡 เหลือง | รออนุมัติ — Admin ตรวจสอบ |
| **Approved** | 🔵 น้ำเงิน | อนุมัติแล้ว — รอนัดหมาย |
| **Appointment** | 🔵 น้ำเงิน | นัดหมายเรียบร้อย — รอเข้าเปลี่ยน |
| **Done** | 🟢 เขียว | เสร็จสิ้น |
| **Rejected** | — | ปฏิเสธ พร้อมระบุเหตุผล |

### ข้อมูลที่ Admin เห็นก่อนอนุมัติ
- ข้อมูลรถ: ทะเบียน / เบอร์รถ / คนขับ / ฟลีท / เลขไมล์
- ข้อมูลยางแต่ละเส้น: ตำแหน่ง / Serial No / สาเหตุ / รูปถ่าย
- ประสิทธิภาพ: ระยะทางใช้ไป / % คงเหลือ / ฿ต่อกิโล จริง vs มาตรฐาน (แดง = เกินมาตรฐาน)

### การอนุมัติ
- อนุมัติ/ปฏิเสธได้ **ทั้งคำขอ** หรือ **รายเส้น** ก็ได้
- เมื่ออนุมัติแล้ว → **นัดหมายวันเข้าเปลี่ยน** → คนขับจะได้รับแจ้ง
- เมื่อเปลี่ยนเสร็จ → กด **Done** เพื่อปิดงาน

## สรุป Flow ทั้งระบบ

```
┌─────────────────────────────────────────────────────────────────┐
│  ADMIN รับยางเข้า Stock                                          │
│  /stock-tire → เพิ่มรายการ (ระบุ PR Code, Serial No, ราคา ฯลฯ) │
└────────────────────────┬────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│  คนขับพบปัญหายาง                                                │
│  /change-tire-request → กรอกทะเบียน + ไมล์ + เลือกยาง + รูป   │
└────────────────────────┬────────────────────────────────────────┘
                         │  ส่งคำขอ
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│  ADMIN พิจารณา                                                   │
│  /requests → ดูข้อมูลครบ → อนุมัติ → นัดหมาย → Done            │
└────────────────────────┬────────────────────────────────────────┘
                         │  เปลี่ยนยางจริง + บันทึกใน ATMS
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│  ระบบ Sync ข้อมูลจาก ATMS (อัตโนมัติทุกวัน / กด Manual)        │
│  /change-history → ประวัติยางอัปเดต + Stock Status อัปเดต       │
└────────────────────────┬────────────────────────────────────────┘
                         │  ครบล็อต PR
                         ▼
┌─────────────────────────────────────────────────────────────────┐
│  ADMIN วิเคราะห์ผลการใช้งาน                                     │
│  /stock-tire → รายงาน PR → ฿/กม. / ประสิทธิภาพ / สาเหตุ        │
│  → ใช้ประกอบการสั่งซื้อยางล็อตถัดไป                            │
└─────────────────────────────────────────────────────────────────┘
```
